## Section Guidelines

### Background
- Explain concepts unfamiliar readers might not know
- Describe current system behavior
- Point to relevant files/functions in the codebase

### Motivation
- Focus on the "why" not the "what"
- Quantify impact if possible (performance, developer time, user complaints)
- Connect to broader project goals

### Proposed Solutions
- Present at least 2 options when feasible
- Include runnable code snippets
- Be objectiveâ€”never label one as "recommended"
- Consider implementation complexity, maintenance burden, and backward compatibility

### Already Answered (Feature Requests)
- Anticipate skepticism and address it proactively
- Document rejected alternatives and why
- Reduces back-and-forth in issue comments
